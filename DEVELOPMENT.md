# A-Plan 开发与维护指南 (极客警示录)

## ⚠️ 版本更新核心红线 (The Golden Rule)

**真相源唯一性**：本项目采用基于 Git Tags 的自动化更新架构。
- **严禁**：在 Pod 本地手动通过 `fs` 修改 `VERSION` 或 `package.json` 的版本号来尝试“对齐”UI。这会导致 Web 后台的“当前版本”与“远程最新版本”逻辑发生严重冲突（显示回退更新或版本号消失）。
- **必须**：所有版本升级必须通过 `git tag -a vX.X.X` 并 `git push --tags` 到 GitHub 远程仓库。只有这样，所有寄生在不同服务器上的 Pod 节点才能通过 Web UI 一键平滑升级。

## 🛠️ v4.2.1 极客特供版 (Ghost Mode) 更新记录

### 1. 幽灵启动逻辑 (Pod 调度防灾)
- **背景**：针对 Pod 环境 3-5 天被调度重启的特性，API 核心必须秒级可用。
- **实现**：引入 `GHOST_MODE=true` 环境变量。
- **坑位**：早期尝试在本地硬编码版本，导致 UI 无法感知。现在已修正为在 master 进程启动时即打通 GitHub 链路。

### 2. Clash 代理分流塔 (Regional Routing)
- **初衷**：Clash 不应在部署后盲目自启夺取全局控制权。
- **优化**：现在 Clash 模块默认保持静默（Idle）。通过 `configs/clash-module.json` 中的 `routing` 表，实现 Provider 级的精准出口控制（如：`gemini -> US`, `claude -> HK`）。
- **避坑**：路由逻辑已从“端口映射”进化为“语义化标签”，UI 上不再展示生涩的端口号。

### 3. 插件彻底卸载 Bug 修复
- **坑位**：之前卸载插件只删文件不删配置，导致重启时系统“自愈”功能触发，因找不到目录而报 `目录名称无效`。
- **修复**：重写 `uninstallPlugin` 逻辑，强制同步清理 `configs/plugins.json`。
