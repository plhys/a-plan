## 🚨 [CRITICAL] 维护者发布纪律 (必须遵守)

### 1. 单一真相来源 (SSOT)
- 修改版本时，**只需且只能**修改根目录的 `VERSION` 文件。Master 进程会自动同步全链路。

### 2. 云端对齐协议
- 每次提交后必须同步创建并推送 Git Tag（如 `v4.2.7`）。
- **流程**: `git add .` -> `git commit` -> `git tag -a vX.X.X` -> `git push origin [branch] --tags`。
- **警告**: 不打 Tag 会导致后台“检查更新”功能失效，造成严重的版本认知差。
