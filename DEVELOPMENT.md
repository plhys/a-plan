## 🚨 维护者血泪踩坑警示录 (后来者必读)

作为 Skywork-Agent，我在 v4.2.4 的极限压榨中踩到了以下三个深坑，请后来者引以为戒：

### 1. 字符串切割的“幻肢痛” (The Split Phantom)
- **坑位**: 在 `request-handler.js` 中盲目重构 `serveStaticFiles`。
- **后果**: 误将 Request 对象当成 Path 字符串传入，导致 `path.split` 触发 `TypeError`，管理后台首页直接 500。
- **警戒**: **永远不要假设参数类型。** 在进行函数重构时，第一步必须确认入参契约。一个小小的类型误判，就能让价值连城的极客 UI 变成一堆废纸。

### 2. JIT 刷新的“惊群效应” (The Thundering Herd)
- **坑位**: 移除后台心跳改为懒加载刷新时，如果没有互斥锁保护。
- **后果**: 高并发下，多个请求会同时发现 Token 到期并同时发起刷新。这会造成凭证文件被写坏，甚至导致多个号同时因重复认证被封。
- **警戒**: **懒加载必须配互斥锁。** 越是“懒”的逻辑，在执行那一瞬间就必须越“严谨”。

### 3. 环境隔离的“色盲”症 (The Environment Color-Blindness)
- **坑位**: 在受限容器（如某些 POD）中进行 Clash 测速。
- **后果**: 宿主机出口策略导致测速包全丢，UI 全屏“飘红”，给人一种“代理全死”的假象。
- **警戒**: **不要迷信测速。** 延迟数字只是参考，API 通了才是真理。在极简环境中，保持逻辑链路的畅通比追求 UI 的“全绿”更重要。
