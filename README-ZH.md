# 🚀 A计划 (A-Plan) v3.0 - 极致性能与高可用 AI 网关

**A计划 (A-Plan)** 是一款专为极客和生产环境设计的 AI 接口转换网关。基于 `AIClient-2-API` 深度重构，旨在提供**毫秒级冷启动速度**、**极致的运行稳定性**以及**自动化的云端配置同步**。

它是 K8s Pod 频繁调度环境下的最佳选择，确保您的 API 服务在任何时候都能“瞬时复活”。

---

## ✨ v3.0 重大更新：模块化与高可用架构

- **【极速与功能平衡】**：支持 `CORE_ONLY` 运行模式。开启后系统进入“高精核心状态”，关闭 UI 看板与冗余统计，仅保留 API 转发、号池管理与**插件式认证引擎**。
- **【插件热插拔 (New)】**：完整找回并优化了原项目的插件系统。支持 AI Monitor (实时监控)、API Potluck (号池管理插件) 等。所有插件均可通过 `configs/plugins.json` 动态控制开关。
- **【强力容灾守护】**：
  - **Master-Worker 级自愈**：主进程实时监控 Worker 状态，崩溃秒级重启。
  - **智能号池容错**：自动感知失效凭证（Key/Token），实现无感自动切号，确保请求成功率。
  - **资源配额自适应**：自动根据 Pod 限制调整 Node.js 运行时参数，防止 OOM 导致的暴力被杀。
- **【异步云端记忆 (Git Sync)】**：优化同步逻辑，将阻塞式 Pull 改为异步后台加载。优先使用本地缓存启动，确保服务“零等待”上线。

## 🛠️ 极速部署 (K8s / Pod 调度环境首选)

针对 Pod 经常被重置但拥有**持久化挂载目录**的环境，推荐以下“秒级复活”方案：

1. **首次部署**：
   ```bash
   git clone https://github.com/plhys/a-plan.git /your/persistent/path/a-plan
   cd a-plan && npm install --production
   ```
2. **一键复活 (start.sh)**：
   每次 Pod 启动或重启时，只需运行根目录的脚本：
   ```bash
   sh start.sh
   ```
   **脚本核心逻辑：**
   - **异步同步**：后台拉取最新 Git 配置，服务启动不等待。
   - **自愈启动**：自动检测进程状态，防重复启动，防死锁。
   - **同步守护**：实时监控 Web UI 的配置变更并自动推送到云端。

## ⚙️ 核心配置开关 (极客选项)

通过设置环境变量控制系统行为：

| 变量名 | 默认值 | 说明 |
| :--- | :--- | :--- |
| `CORE_ONLY` | `false` | **开启极速模式**。为 `true` 时关闭 Dashboard 和高级统计，仅保留 API 转发。 |
| `AUTO_SYNC` | `true` | 是否开启 Git 云端自动同步。 |
| `WORKER_COUNT` | `1` | 工作进程数量，建议设为 CPU 核心数或 `auto`。 |

## 💾 云端持久化设置 (Git Sync)

为了彻底解决 Pod 重置导致数据丢失的问题，请务必启用 Git 同步：

1. **准备仓库**：在 GitHub 上创建一个 **私有仓库**，并生成具有写入权限的 **Personal Access Token (PAT)**。
2. **配置连接**：在 A计划 后台【配置管理】 -> 【云端持久化】中启用，并填入地址：
   `https://<YOUR_TOKEN>@github.com/<USERNAME>/<REPO_NAME>.git`
3. **效果**：所有号池、API Key 的修改都会在秒级自动同步到云端。Pod 被调度后，`start.sh` 会自动拉回所有数据，**真正做到零配置迁移**。
